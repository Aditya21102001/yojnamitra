# Deploying YojanaMitra

Dockerfiles for all three services live in `ai/`, `api/`, and `web/`. This app
runs a **local LLM**, which shapes deployment — the two real constraints are
called out below so there are no surprises.

## Build & run as containers (local Docker)

```bash
docker build -t ym-ai  ./ai
docker build -t ym-api ./api
docker build -t ym-web ./web

docker compose up -d postgres            # Postgres 16 (see docker-compose.yml)

# GenAI service — needs your host's Ollama:
docker run -p 8000:8000 -e OLLAMA_HOST=http://host.docker.internal:11434 ym-ai
# API — talks to the GenAI service and Postgres:
docker run -p 8080:8080 \
  -e YOJANAMITRA_AI_BASE_URL=http://host.docker.internal:8000 \
  -e DB_ENGINE=postgres -e DB_HOST=host.docker.internal -e DB_PASSWORD=postgres \
  -e YOJANAMITRA_JWT_SECRET=change-me ym-api
# Web (nginx):
docker run -p 4200:80 ym-web
```

## Two honest constraints

1. **The LLM (Ollama) needs real compute.** Free web tiers (Render/Railway free
   dynos) can't run Ollama — no GPU, too little RAM. Options:
   - self-host Ollama on a VM/box and point `OLLAMA_HOST` at it, **or**
   - swap `ai/ollama_client.py` to a free *hosted* LLM API (e.g. Groq or
     OpenRouter free models). It only exposes three functions — `embed()`,
     `generate()`, `generate_json()` — so the rest of the app is unaffected.

2. **The Angular API base URL is `http://localhost:8080/api`** (in
   `web/src/app/api.ts` and `auth.ts`). Point these at your deployed API origin
   before building the web image, or externalize them to a runtime config.

## Render blueprint

[`render.yaml`](render.yaml) deploys the **api** (Docker) + **web** (static) +
a free **Postgres**. You still supply an Ollama host (constraint #1) and the
GenAI service URL. JWT secret is generated by Render.
